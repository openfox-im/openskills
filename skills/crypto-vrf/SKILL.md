This skill implements a Verifiable Random Function (VRF) using DLEQ (Discrete Log Equality) proofs on the Ristretto255 group, compatible with the Schnorrkel 0.11.x VRF specification. A VRF takes a secret key and an input message and produces a deterministic pseudorandom output along with a proof that the output was correctly computed. Anyone with the corresponding public key can verify the proof without learning the secret key. Use this for leader election, lottery systems, or any protocol requiring publicly verifiable randomness.

The prove backend accepts a 32-byte seed (from which a keypair is derived) and an input message, returning a 32-byte VRF output and a 64-byte DLEQ proof. The domain separator is "TOS-VRF-v1". The verify backend takes a public key, input message, VRF output, and proof, and returns whether the proof is valid. The VRF output is derived by hashing the Ristretto255 point to 32 bytes, ensuring uniform distribution in the output space.

Header reference: `lib/include/at/crypto/at_vrf.h`. The implementation uses the same Ristretto255 group infrastructure as the Ed25519 and Schnorr skills. Compatibility with Schnorrkel 0.11.x means that proofs generated by this skill can be verified by Substrate/Polkadot-ecosystem VRF verifiers and vice versa. The DLEQ proof ensures that the same discrete log relationship holds between the basepoint/public-key pair and the input-point/output-point pair, preventing the prover from biasing the output.
